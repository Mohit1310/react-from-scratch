Difference between a Library and Framework?

A library is like a toolbox filled with various tools. Each tool has a specific function, like a hammer, a screwdriver, or a wrench. When you want to build something, you can take out the tools you need from the toolbox and use them individually to create your project. You have the freedom to choose which tools to use and how to use them, allowing you to build things your own way.

On the other hand, a framework is like a blueprint or a set of guidelines for building something specific, like a house. It provides you with a structured plan, telling you what materials to use, where to put each part, and how to assemble everything together. It's like following a step-by-step instruction manual that ensures all the pieces fit together perfectly. The framework helps you build the project more efficiently and ensures consistency, but it might limit some of your creative freedom compared to using a toolbox full of various tools.

What is CDN and why do we use it?

It is a content delivery network. We use it for faster load times, scalability, cost savings, etc. It is a powerful tool to accelerate website loading times and experience

Why React is known as React?

React is known as "React" because its name comes from its core concept of "reacting" to changes in data and updating the user interface accordingly.

What is crossorigin in script tag?

The `crossorigin` attribute in the script tag allows loading scripts from different domains securely. It prevents browser security issues when fetching scripts from external sources by enforcing proper permissions.

async vs defer Attributes for JavaScript via CDN

Aspect async Attribute defer Attribute
Parsing and Execution Downloads the script asynchronously while parsing HTML, then executes it immediately after downloading, potentially blocking HTML parsing Downloads the script asynchronously while parsing HTML, then defers execution until the HTML parsing is complete
Execution Order Execution order may not be guaranteed, leading to potential dependency issues Execution order is preserved, making it suitable for scripts with dependencies
DOM Interaction May lead to scripts executing before the DOM is fully constructed Script execution waits until the DOM is fully constructed
Usage Use for scripts that are independent and don't rely on DOM elements or other scripts Use for scripts that require access to DOM elements or have dependencies on other scripts
Recommended Recommended for non-blocking scripts that can operate independently Recommended for scripts that need to be executed in a specific order or interact with the DOM
